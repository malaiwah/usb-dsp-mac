Protocol clarification from this capture:
  Direction a2/53 = READ (host queries, device responds with current value)
  Direction a1/51 = WRITE (host sends new value, device acks with 0-length data)
  This is the same a1/51 pair used for firmware blocks — it is a general "write" indicator,
  not exclusive to firmware mode.

Write command format (0x1fNN, dir=a1):
  data[0..1]: 01 00 (constant)
  data[2..5]: value (LE uint32) — 0 = reset/default
  data[6]:    00
  data[7]:    sub-index (varies per cmd: 0x12 for 0x1f07, 0x0f for 0x1f06, etc.)
